    üìå What is Flower?

    Flower is a real-time web-based monitoring tool for Celery. It provides a dashboard to inspect, control, and monitor Celery workers and tasks.

    It‚Äôs incredibly useful for production environments where observability and quick debugging of asynchronous tasks are critical.

    üöÄ Features

    Feature Description
    Task monitoring See live task statuses: received, started, succeeded, failed, retried
    Worker monitoring Track online/offline workers and their resource usage
    Task details Inspect input arguments, results, exception traceback
    Control workers Restart, shutdown, or revoke tasks from the UI
    Broadcast commands Broadcast shutdown/revoke/purge commands to multiple workers
    Real-time updates Uses WebSockets for live updates

    üîß How to Install

    pip install flower

    üèÅ How to Run

    With Celery (basic usage):

    celery -A your_project flower

    Specify port (optional):

    celery -A your_project flower --port=5555

    Then visit: http://localhost:5555

    üõ† Common Configuration Options

    Option Description
    --port=5555 Set custom port
    --broker=amqp://... Set the broker URL (RabbitMQ/Redis)
    --basic_auth=user:pass Secure the dashboard
    --url_prefix=/flower Mount Flower behind a reverse proxy path

    üîí Securing Flower (Optional)

    You can add basic authentication:

    celery -A your_project flower --basic_auth=user:password

    üìä Why Use Flower?

    • Visual overview of all tasks and their states
    • Easier debugging and tracking of failed tasks
    • Useful during development, staging, and production
    • Lightweight and easy to integrate

    üìâ Limitations

    • Not meant for deep log aggregation or alerting (use with Prometheus/Grafana for full observability)
    • Not designed for heavy customization
