    Overview

    The Integration-Framework is a modular and extensible system designed to facilitate seamless communication between various banking and enterprise services. It includes API wrappers, data parsers, configuration management, and testing utilities to ensure smooth integration and execution of financial and authentication-related workflows.

    Project Structure

    API Wrappers (api/)

    Contains Python modules that wrap external services for ease of use:

    api_wrapper.py – General API wrapper for handling requests.

email_wrapper.py – Manages email-based interactions.

finacle_wrapper.py – Interfaces with the Finacle Core Banking System.

ldap_wrapper.py – Handles LDAP authentication and user management.

sms_wrapper.py – Sends and receives SMS messages.

    CAML Types (caml_types/)

    Defines structured types for handling API requests and responses:

    api_types.py – Defines API-related data structures.

config_types.py – Holds configuration-related types.

input_entity_types.py – Manages input entities for various services.

request_config_types.py – Specifies request configurations.

response_entity_types.py – Defines expected API response formats.

    Finacle XML Requests (finacle/)

    Includes pre-defined XML templates for Finacle banking operations:

    CIFAdd.xml – Adds a Customer Information File (CIF).

DedupCheckCIF.xml – Performs duplicate checks on CIFs.

SBAActAdd.xml – Adds Savings Bank Accounts (SBA).

SVSSignatureAdd.xml – Stores signature verification data.

VerifyCIF.xml – Validates customer CIF details.

    Finacle Response Parsers (finacle_helper/)

    Handles the processing of Finacle API responses:

    cif_add_parser.py – Parses responses for CIF Add operations.

dup_check_parser.py – Parses duplicate check results.

generic_response_parser.py – General-purpose response parser.

    Configuration & Input Files

    config.toml – Holds framework-wide configuration settings.

input.toml & inputdata.toml – Provide sample input data.

    Scripts (scripts/)

    Contains test scripts and utilities for interacting with APIs:

    _test_api.py – Tests general API functionality.

_test_aml_ind.py – Tests Anti-Money Laundering (AML) functions.

_test_digio.py – Validates integration with Digio services.

_test_zeep.py – Tests Zeep SOAP-based API clients.

pipe-line_finacle_test.py – Automates Finacle API testing.

    Task Helper (task_helper/)

    api_clients.py – Manages API clients.

api_factory.py – Factory pattern for creating API instances.

function_runner_mixin.py – Provides function execution utilities.

    Testing (tests/)

    Includes unit tests for various components:

    test_cif_add.py – Tests CIF Add operation.

test_dup_check.py – Tests duplicate CIF check.

test_email.py – Tests email functionality.

test_ldap.py – Validates LDAP authentication.

test_sba_add.py – Tests Savings Bank Account (SBA) addition.

test_sms.py – Tests SMS service interactions.

test_svs_add.py – Tests SVS signature addition.

    Utilities (utils/)

    interpolation_utils.py – Manages dynamic value interpolation.

utils.py – Contains general utility functions.

    Workspace & Dependencies

    Integration-Framework.code-workspace – VS Code workspace settings.

    Justfile – Task automation script (similar to Makefile).

pyproject.toml – Python project dependencies and configuration.

uv.lock – Dependency lockfile for dependency management.

    Key Features

    ✅ API Wrappers – Seamless integration with Finacle, LDAP, and SMS services.
✅ XML-Based Requests – Predefined Finacle XML request templates.
✅ Response Parsers – Efficient handling of Finacle responses.
✅ Configuration-Driven – Uses TOML files for easy customization.
✅ Testing Suite – Comprehensive unit tests for robust API validation.
✅ Task Automation – Supports Justfile for easy task execution.
