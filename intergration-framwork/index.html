    Integration Framework

    This document outlines the process of integrating with Any SOAP-based services using Pydantic models. The solution automates the generation of Pydantic models from the WSDL file provided by Finacle, ensuring robust data validation, simplifying payload handling, and accelerating development.

    ** Important **

    2. Replace with your WSDL URL

    Overview

    Objective

    1. Automate the generation of Pydantic models from Finacle's WSDL file.
    2. Validate request and response payloads using Pydantic's data validation capabilities.
    3. Rapidly implement Finacle services by reusing generated models.

    Workflow

    1. Fetch the WSDL file from the Finacle SOAP endpoint.
    2. Dynamically generate Pydantic models for all SOAP operations (input and output).
    3. Use the generated models to validate payloads and interact with Finacle services.
    4. Copy the standalone models into caml_types/input_entity_types.py for reuse across the system.

    Benefits

    1. Type Safety: Ensures data integrity by validating request and response payloads.
    2. Rapid Development: Reduces manual effort by automating model generation.
    3. Reusability: Generated models can be reused across multiple services.
    4. Standardization: Provides a consistent way to handle Finacle SOAP APIs.

    About Generator Code

    Fetching the WSDL File

    The WSDL file is fetched from the Finacle SOAP endpoint. This file describes the available SOAP operations, their input parameters, and output structures.

    Code

    from zeep import Client
# Finacle SOAP endpoint
wsdl_url = "https://finacle-endpoint.com/service?wsdl"
client = Client(wsdl_url)

    Generating Pydantic Models

    Using the WSDL file, we dynamically generate Pydantic models for each SOAP operation. These models represent the input and output structures of the operations.

    Code

    from pydantic import BaseModel, Field, create_model
from typing import Dict, Type

# Dictionary to store generated models
pydantic_models: Dict[str, Type[BaseModel]] = {}

# Function to map SOAP types to Python types
def map_soap_type_to_python_type(soap_type):
    if soap_type == "xsd:string":
        return str
    elif soap_type == "xsd:int":
        return int
    elif soap_type == "xsd:float":
        return float
    elif soap_type == "xsd:boolean":
        return bool
    elif soap_type == "xsd:dateTime":
        return str
    else:
        return str  # Default to string for unknown types

# Function to create Pydantic models
def create_pydantic_model(name: str, elements: Dict[str, str]) -> Type[BaseModel]:
    annotations = {
        element: (map_soap_type_to_python_type(type_), Field(..., description=f"Field {element}"))
        for element, type_ in elements.items()
    }
    return create_model(name, **annotations)

# Iterate over SOAP operations and generate models
for service in client.wsdl.services.values():
    for port in service.ports.values():
        operations = port.binding._operations
        for operation_name, operation in operations.items():
            # Generate input model
            input_elements = {element[0]: element[1].type for element in operation.input.body.type.elements}
            input_model = create_pydantic_model(f"{operation_name}Input", input_elements)
            pydantic_models[f"{operation_name}Input"] = input_model

            # Generate output model
            output_elements = {element[0]: element[1].type for element in operation.output.body.type.elements}
            output_model = create_pydantic_model(f"{operation_name}Output", output_elements)
            pydantic_models[f"{operation_name}Output"] = output_model

    Key Features

    1. Automation:
      • Dynamically generates Pydantic models from the WSDL file.
      • Reduces manual effort and ensures consistency.
    2. Validation:
      • Validates request and response payloads using Pydantic.
      • Ensures data integrity and type safety.
    3. Reusability:
      • Generated models are stored in caml_types/input_entity_types.py for reuse.
      • Promotes a standardized approach to handling Finacle services.
    4. Scalability:
      • Can handle multiple Finacle services and operations.
      • Easily extendable to support additional SOAP APIs.

    Conclusion

    This solution provides a robust and efficient way to integrate with Finacle's SOAP services. By automating the generation of Pydantic models and validating payloads, it accelerates development, ensures data integrity, and promotes reusability across the system. This approach is particularly beneficial for rapidly implementing Finacle services while maintaining high standards of code quality and reliability.
